야외 지형을 만들 수 있는 기능
- 높낮이, 날카로움, 부드러움, 평편함 편집
- 특정 모양으로 조각
- 머터리얼 부여

컴포넌트
- 충돌처리, 렌더링, 편집의 기본 단위
- 정사각형
- 컴포넌트의 크기는 모두 같음
- 렌더링, Visibility Check, Collision의 기본 단위
- 적을수록 성능 향상(드로우 콜 수 에 영향을 미치기 때문)

=> 모드 - 랜드 스케이프
=> 노란선은 하나의 컴포넌트를 의미한다.
=> 디폴트는 정사각형이므로, 다른 모형을 하고 싶은 경우에는 컴포넌트 수를 변경해야 한다.

섹션
- 컴포넌트의 디테일 수준을 정하는 단위
- 하나의 컴포넌트는 하나 이상의 섹션으로 구성될 수 있음
- (그림 참고)
- 컴포넌트 수를 줄이면서도 디테일을 유지할 수 있게 해 줌(ex: Height Map)

쿼드 - 가장 작은 단위
- 섹션의 디테일 수준을 정하는 단위
- 하나의 섹션은 하나의 쿼드로 구성될 수 있음
- 쿼드의 기본 크기는 어떤 지형에 상관없이 모두 같음
=> 쿼드의 크기에 따라 컴포넌트의 크기가 변한다.
- 하이트맵의 한 픽셀과 1:1로 대응

컴포넌트, 섹션, 쿼드 개수를 조절하여 여러 가지 옵션으로 지형 생성 가능
- 컴포넌트 증가 -> LOD 변화 빠름, 드로우 콜 증가
- 섹션 증가 -> 컴포넌트 개수 감소, LOD 변환 느림
- 맵의 특성에 따라 신중히 결정!

#Lecture 11
■ 라이트 모빌리티
=> 게임의 성능을 저하시키는 많은 요소 중 하나는 Lighting 연산

스태틱(Static)
- 실행 후 아무것도 바꿀 수 없는 라이트
- 라이트맵 안에서만 계산되며, 한 번 처리된 이후로는 퍼포먼스에 영향을 끼치지 않음
- 퀄리티 중간, 변화 가능성 최저, 퍼포먼스 비용 최저
- 정적인 광원이 존재할 경우 사용함(햇빛, 고정된 전등, ...)
- 퍼포먼스 대비 높은 퀄리티를 얻을 수 있음(즉, 게임 실행 이전에 미리 광원 관련 연산을 수행하여 실행 시에는 광원 관련 연산이 없음)

스테이셔너리(Stationary)
- 실행 후 색이나 밝기 등을 바꿀 수 있는 라이트
- 움직이거나 회전은 불가능
- 한 위치에 머물 되, 밝기나 색과 같은 것은 바꿀 수 있는 라이트
- 퀄리티 최고, 변화가능성 중간, 퍼포먼스 비용 중간
- 쉐도우 볼륨을 유지하기 때문에 움직이는 오브젝트도 그림자의 영향을 받을 수 있음

무버블(Movable)
- 실행 후 색이나 밝기 등을 바꿀 수 있는 라이트
- 움직이거나 회전 가능
- 완벽히 동적인 빛과 그림자를 드리우며, 위치나 방향, 색, 밝기, 감쇄, 반경 등 그 모든 프로퍼티를 변경할 수 있음
- 무버블 라이트가 도달하는 오브젝트의 개수에 따라 퍼포먼스가 크게 요동침
- 손전등, 움직이는 전등

오브젝트 모빌리티와 관계를 각자 이해해 보자
- 라이트 : 무버블, 오브젝트 : 무버블
- 라이트 : 스테이셔너리, 오브젝트 : 스태틱
- 라이트 : 스태틱, 오브젝트 : 스테이셔너리
