■ Material(Inputs 2)
- 광원과 관련된 연산은 반드시 베이스컬러의 입력이 있어야만 한다.
- 베이스 컬러가 검정색이면 아무런 영향이 없기 때문

오파시티 마스크
- 1이면 그려지고, 0이면 안그려짐
- 철조망 등에 사용
- Blend를 사용하지 않고 Depth Buffer를 이용

=> 오파시티는 그림자가 생기지 않음
=> Alpha 값과 Depth Buffer는 상반된 알고리즘을 사용

노멀
- 해당 픽셀의 노멀 방향을 변경
- 텍스처로 표면 굴곡을 표현할 때 사용

월드 포지션 오프셋
월드 디스플레이스먼트
테셀레이션 멀티플라이어
- 버텍스의 위치를 변경
- 버텍스의 위치 변경을 통해 다양한 표현이 필요할 때 사용
- 즉, Displacement Mapping

■ Displacement Mapping
셰이더 내에서 버텍스 좌표를 바꾸는 기법
- 노멀 맵과 다름
- 실제 좌표가 변경됨

정밀한 Displacement Mapping
- 버텍스 수가 많아야 함
- 원본 데이터의 버텍스 수가 적다면??

=> 월드 디스플레이스먼트의 입력이 없다면 정점이 그대로 출력 됨
=> 입력이 있다면, WorldPosition의 값만큼 정점에 더해져서 출력됨
=> 각각의 정점을 노멀 벡터의 방향으로 신축
=> VertexNormalWS 노드 : Tangent Space 상에서 정점당 주어진 노말을 가져오는 노드

#Lecture 4_2
실습 : 꿈틀거리는 무언가를 만들어 보자
- BlackBody : 온도를 받아 색상을 계산하는 노드
- 꿈틀거리는 기능을 위해 실시간(Time)으로 Noise를 생성하는 노드가 필요함

=> Noise 노드는 대리석 같은 것을 표현할 때도 사용한다.
=> 원기둥의 기둥과 같은 곳에 텍스처를 적용하면 끊어져서 부자연스러울 수 있기 때문에 Tiled를 지원하는 텍스쳐를 사용하자! (Noise는 텍스처가 아니므로 자연스럽게 연결됨)
=> 입력되는 조명들의 영향을 받지 않기 위해서는 Shading Model을 "Unlit"으로 변경하자!
=> Noise Node는 -1.0 ~ 1.0의 값을 출력하므로, 적절히 필터링 해서 사용하자(0.0 ~ 1.0으로 쓸려면 +1.0을 하고 2로 나누자)
=> Noise Node는 3D Texture 샘플링 기법을 이용하여 임의의 값을 생성해 냄, 이 때의 Seed는 볼륨 매트릭 데이터의 한점을 기반으로 한다.
